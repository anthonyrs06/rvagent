# Deploying Resume Vault on Render

Staged launch: deploy to `*.onrender.com` first, then add custom domain and Turnstile.

## Prerequisites

- GitHub repo: [anthonyrs06/rvagent](https://github.com/anthonyrs06/rvagent)
- Render account with Starter plan access (persistent disk required)
- PostHog project with session replay enabled
- (Phase 2) DNS access for `anthonysadarangani.com`
- (Phase 2) Cloudflare Turnstile widget

## Phase 1 — Initial deploy (`*.onrender.com`)

### 1. Push code to GitHub

```bash
git remote add origin https://github.com/anthonyrs06/rvagent.git   # or set-url if exists
git push -u origin main
```

### 2. Create Render Blueprint

1. Render Dashboard → **New** → **Blueprint**
2. Connect **anthonyrs06/rvagent** (branch `main`)
3. Apply the root [`render.yaml`](render.yaml)

### 3. Set secret environment variables

**Set these before the first build completes** (Vite bakes `VITE_PUBLIC_*` at build time):

| Variable | Value | Notes |
|---|---|---|
| `OWNER_PASSWORD` | your choice | Min 12 characters; `/admin` login |
| `VITE_PUBLIC_POSTHOG_TOKEN` | `phc_…` | Project API key — build-time |
| `VITE_PUBLIC_POSTHOG_HOST` | `https://us.i.posthog.com` | Pre-set in blueprint |
| `POSTHOG_TOKEN` | same `phc_` value | Server mirror |
| `POSTHOG_HOST` | `https://us.i.posthog.com` | Pre-set in blueprint |
| `POSTHOG_PERSONAL_API_KEY` | `phx_…` | Personal API key for HogQL |
| `POSTHOG_PROJECT_ID` | numeric ID | See below |

**Discover `POSTHOG_PROJECT_ID`:**

```bash
curl -s -H "Authorization: Bearer $POSTHOG_PERSONAL_API_KEY" \
  https://us.posthog.com/api/projects/ | jq '.results[0].id'
```

Skip Turnstile for Phase 1 — the gate works without it.

`SESSION_SECRET` and `IP_HASH_SALT` are auto-generated by the blueprint.

### 4. Confirm persistent disk

In Render → your service → **Disks**: verify `resume-vault-data` is mounted at `/app/data`.

### 5. Smoke test

On `https://<service-name>.onrender.com`:

- [ ] `GET /health` returns `{"ok":true}`
- [ ] `/admin` login with `OWNER_PASSWORD`
- [ ] Upload a PDF → create a share link → open `/r/:token`
- [ ] Watermarked pages render in the canvas viewer
- [ ] Tile URL without session cookie → 401

### 6. PostHog setup

In PostHog → Project Settings:

- Enable **Session replay**
- Add `*.onrender.com` to **Authorized URLs**
- Confirm live events: `resume_opened_client`, `page_viewed`, `page_dwell`
- Confirm session replays show masked text and blank canvas (resume content not visible)
- Admin dashboard "Opens — last 30 days" chart should use PostHog (not first-party fallback)

## Phase 2 — Custom domain + Turnstile (`anthonysadarangani.com`)

### 7. Add custom domain in Render

1. Service → **Settings** → **Custom Domains**
2. Add `anthonysadarangani.com`

### 8. Configure DNS

At your domain registrar:

- Create **A record** `@` → IP shown in Render Dashboard
- Remove any stale **AAAA** records for the apex
- Wait for TLS verification (Render issues the certificate automatically)

Optional: add `www` — Render can redirect apex ↔ www.

### 9. Cloudflare Turnstile

1. Create a Turnstile widget
2. Allowed domains: `anthonysadarangani.com` and `*.onrender.com`
3. Set in Render:
   - `TURNSTILE_SITE_KEY`
   - `TURNSTILE_SECRET_KEY`
4. Redeploy

### 10. PostHog authorized URLs

Add `anthonysadarangani.com` to PostHog authorized URLs.

## Pre-launch verification checklist

- [ ] `/health` returns 200
- [ ] `/admin` requires password; wrong password rate-limited
- [ ] Upload PDF → rasterize → create link → viewer gate → watermarked tiles
- [ ] Tile URL without session cookie → 401
- [ ] Revoke link → tiles 401 immediately
- [ ] Password gate rejects wrong password
- [ ] Redacted zone + click-to-reveal logged
- [ ] Turnstile challenge appears on gate (Phase 2)
- [ ] Link preview test: paste URL in Slack/iMessage — preview must **not** show recipient name
- [ ] Custom domain `anthonysadarangani.com` serves HTTPS (Phase 2)
- [ ] Disk persists data across redeploy (upload resume, redeploy, resume still there)
- [ ] `/admin` sessions do **not** appear in PostHog replays

## Architecture notes

- **Single instance only** — persistent disk prevents horizontal scaling
- **Brief downtime on deploy** — acceptable for a personal resume vault
- **Ephemeral filesystem** — only `/app/data` persists; SQLite + PDFs must live there
- **Bind address** — `HOST=0.0.0.0`, `PORT` injected by Render

## Troubleshooting

| Symptom | Fix |
|---|---|
| Boot fails with env validation error | Set all required secrets in Render Dashboard |
| PostHog replay missing | Ensure `VITE_PUBLIC_POSTHOG_TOKEN` was set **before** build; trigger manual redeploy |
| Admin login disabled | `OWNER_PASSWORD` must be ≥ 12 characters |
| Data lost after deploy | Confirm persistent disk mounted at `/app/data` |
| Turnstile fails | Add production hostname to widget allowed domains |
